---
title: "创建工单"
description: "如何创建、分配、安排和跟踪服务工单。"
---

**工单**代表对电梯的一次服务访问。每次技术员去现场,LiftAuth 中都应该有一个工单来记录。每个工单恰好关联一部电梯。

## 三种工单类型

<CardGroup cols={1}>
  <Card icon="calendar-check">
    ## Maintenance

    有计划的例行检查。通常由 Maintenance Plan 自动创建。用于定期安排的服务。
  </Card>

  <Card icon="badge-x">
    ## Breakdown

    紧急出勤。当电梯出现故障或不安全且需要立即处理时使用。
  </Card>

  <Card icon="badge-alert">
    ## Repair

    修复特定已知故障的访问。当您要跟进已报告的 issue 时使用。
  </Card>
</CardGroup>

## 工单状态

| 状态 | 含义 |
| --- | --- |
| **Scheduled** | 已分配技术员并确认日期/时间 |
| **Work Done** | 技术员已在现场完成工单。[Record](/admin/records) 已创建 |
| **Signed** | Building Manager 已对记录签字 |
| **Closed** | Admin 已审核并关闭工单 |

## 创建新工单

<Steps>
  <Step title="进入 Jobs">
    在左侧边栏点击 **Jobs**。
  </Step>
  <Step title="点击 Schedule Job">
    点击 **Schedule Job** 按钮并选择工单类型 — Maintenance、Breakdown 或 Repair。
  </Step>
  <Step title="选择电梯">
    选择需要维护的电梯。每个工单仅涵盖一部电梯。
  </Step>
  <Step title="分配技术员">
    选择将执行工作的技术员。
  </Step>
  <Step title="设置日期和时间">
    输入计划日期和预期时间窗口。如果输入错误也不必担心,您可以[稍后修正](/admin/jobs#rescheduling)。
  </Step>
  <Step title="Save">
    点击 **Save**。技术员将在其移动应用中看到该工单。
  </Step>
</Steps>

<Tip>
  如果您已为电梯设置了 [Maintenance Plan](/admin/maintenance-plans),则每个周期都会自动创建 maintenance 工单 — 您只需分配技术员并确认时间。
</Tip>

## 关闭已完成的工单 {#closing-a-job}

一旦技术员完成且 Building Manager 已签字,该工单将出现在您的 Inbox 中,准备关闭。

<Steps>
  <Step title="打开工单">
    在 Inbox 中找到它,或在 Jobs 列表中找到状态为 **Signed** 的工单。
  </Step>
  <Step title="审核记录">
    检查检查清单结果、照片和签名。
  </Step>
  <Step title="关闭工单">
    点击 **Close job** 并确认。工单状态变为 **Closed**。
  </Step>
</Steps>

<Tip>
  如果 Building Manager 不在现场签字,技术员可以通过电子邮件向他们发送签字链接。一旦他们完成,工单将移至 **Signed**。在 [Records](/admin/records#sending-a-signing-link) 中了解如何操作。
</Tip>

## 重新安排或取消工单 {#rescheduling}

<Warning>
  工单可以重新安排或取消,但已取消的工单无法重新开启。如果访问仍需要进行,请创建新工单。
</Warning>

1. 从 Jobs 列表打开工单。
2. 点击 **Reschedule** 或 **Cancel job** 并确认。

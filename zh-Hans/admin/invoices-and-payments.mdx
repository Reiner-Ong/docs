## 发票如何创建

创建发票有两种方式:

<CardGroup cols={2}>
  <Card icon="newspaper">
    ## 通过 Maintenance Plan

    [Maintenance Plan](https://liftauth.mintlify.app/admin/maintenance-plans) 可以生成 Draft Invoices,需要**批准**后才能转换为电梯的实际发票。\
  </Card>

  <Card icon="plus">
    ## 手动

    如果没有 maintenance plan,您可以自己创建发票。前往侧边栏中的 **Invoices** → **New invoice**。填写 building manager、发票日期、到期日期和明细项目。
  </Card>
</CardGroup>

## 发票草稿 — 发送前审核

从 [Maintenance Plans](https://liftauth.mintlify.app/admin/maintenance-plans) 完成工单后生成的发票以 **draft** 状态开始,并出现在您的 Inbox 中。在您批准之前,不会向 building manager 发送任何内容。

**草稿状态:**

| 状态 | 含义 |
| --- | --- |
| **Pending** | 在 Inbox 中等待您审核 |
| **Finalized** | 您已批准 — 发票已创建,准备发送 |
| **Rejected** | 您已拒绝 — 不会发送 |

### 批准或拒绝草稿

<Steps>
  <Step title="从 Inbox 打开草稿">
    点击发票草稿卡片打开它。
  </Step>
  <Step title="检查详细信息">
    审核 building manager、明细项目、总金额和到期日期。
  </Step>
  <Step title="批准或拒绝">
    点击 **Approve** 以将其最终确定并创建发票。如果有错误请点击 **Reject** — 草稿将被标记为已拒绝,不会发送。
  </Step>
</Steps>

<Warning>
  已拒绝的草稿会保留在记录中,但无法编辑或重新发送。如有需要,必须手动创建新发票。
</Warning>

## 发送已最终确定的发票

<Steps>
  <Step title="打开发票">
    前往侧边栏中的 **Invoices** 并点击发票。
  </Step>
  <Step title="点击 Send invoice">
    点击 **Send invoice** 并确认收件人的电子邮件地址。
  </Step>
  <Step title="确认">
    building manager 将通过电子邮件收到发票。
  </Step>
</Steps>

<Tip>
  发票发送到 building manager 的主要联系电子邮件。请确保这在其 [Building Manager 记录](/zh-Hans/admin/building-managers) 中是最新的。
</Tip>

## 记录付款

当 building manager 付款时,在 LiftAuth 中记录它以保持记录最新。

<Steps>
  <Step title="打开发票">
    在 **Invoices** 列表中找到它并点击它。
  </Step>
  <Step title="点击 Record payment">
    点击 **Record payment**,输入金额和收到的日期,然后保存。
  </Step>
</Steps>

<Tip>
  侧边栏中的 **Payments** 部分为您提供了所有已记录付款的完整列表。
</Tip>

## 删除发票

<Warning>
  发票无法删除。一旦创建了发票,它将永久保留在记录中。这是为了确保您的账单历史保持完整和准确,以便合规和审计目的。
</Warning>

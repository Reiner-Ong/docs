## 发票是如何创建的

发票可以通过两种方式创建:

<CardGroup cols={2}>
  <Card icon="newspaper">
    ## 通过 Maintenance Plan

    [Maintenance Plan](https://liftauth.mintlify.app/admin/maintenance-plans) 可以生成需要**批准**才能转换为电梯实际发票的发票草稿。\
  </Card>

  <Card icon="plus">
    ## 手动

    如果没有 maintenance plan,您可以自己创建发票。前往侧栏中的 **Invoices** → **New invoice**。填写 building manager、发票日期、到期日和行项目。
  </Card>
</CardGroup>

## 发票草稿——发送前审核

由 [Maintenance Plans](https://liftauth.mintlify.app/admin/maintenance-plans) 任务完成而生成的发票将以**草稿**形式开始,并出现在您的 Inbox 中。在您批准之前,不会向 building manager 发送任何内容。

**草稿状态:**

| 状态 | 含义 |
| --- | --- |
| **Pending** | 在 Inbox 中等待您审核 |
| **Finalized** | 您已批准——发票已创建并准备发送 |
| **Rejected** | 您已拒绝——不会发送 |

### 批准或拒绝草稿

<Steps>
  <Step title="从您的 Inbox 打开草稿">
    点击发票草稿卡片将其打开。
  </Step>
  <Step title="检查详细信息">
    审核 building manager、行项目、总金额和到期日。
  </Step>
  <Step title="批准或拒绝">
    点击 **Approve** 完成并创建发票。如果有错误,点击 **Reject** ——草稿将被标记为已拒绝,不会发送。
  </Step>
</Steps>

<Warning>
  被拒绝的草稿会保留在记录中,但无法编辑或重新发送。如有需要,需要手动创建新发票。
</Warning>

## 发送已完成的发票

<Steps>
  <Step title="打开发票">
    前往侧栏中的 **Invoices** 并点击该发票。
  </Step>
  <Step title="点击 Send invoice">
    点击 **Send invoice** 并确认收件人的电子邮件地址。
  </Step>
  <Step title="确认">
    building manager 将通过电子邮件收到发票。
  </Step>
</Steps>

<Tip>
  发票发送到 building manager 的主要联系电子邮件。请确保其在 [Building Manager record](/admin/building-managers) 中是最新的。
</Tip>

## 记录付款

当 building manager 付款时,在 LiftAuth 中记录以保持您的记录最新。

<Steps>
  <Step title="打开发票">
    在 **Invoices** 列表中找到并点击它。
  </Step>
  <Step title="点击 Record payment">
    点击 **Record payment**,输入金额和收到日期,然后保存。
  </Step>
</Steps>

<Tip>
  侧栏中的 **Payments** 部分为您提供所有已记录付款的完整列表。
</Tip>

## 删除发票

<Warning>
  发票无法删除。一旦创建发票,就会永久保留在记录中。这是为了确保您的账单历史记录对合规和审计目的保持完整和准确。
</Warning>

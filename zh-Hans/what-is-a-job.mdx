# 工单

工单是对一部电梯所做的一项工作。每个工单都属于一部特定的电梯,有一个类型,并从开始到结束经过相同的步骤。一旦关闭,它就成为该电梯永久历史记录的一部分。

## 工单是什么

工单是对特定电梯的一个工作单元。每个工单:

- 属于一部电梯。
- 有一个类型 — **Maintenance**、**Breakdown** 或 **Repair**。
- 经历与其他工单相同的步骤。
- 一旦安排,就会分配给一名技术员。
- 关闭后成为电梯历史记录的一部分。

已取消的工单不会保存到电梯的历史记录中。

## 工单如何流转

一个工单经过五个步骤:

**Open → Scheduled → Work Done → Signed → Closed**

| 步骤 | 含义 |
| --- | --- |
| Open | 已创建,但还没有技术员或日期。 |
| Scheduled | 已设置技术员和时间。准备开始工作。 |
| Work Done | 技术员已完成现场工作。等待客户签字。 |
| Signed | 楼宇代表已在工单上签字。 |
| Closed | 管理员已审核并关闭工单。它现在是电梯历史记录的一部分。 |
| Cancelled | 该工单不会执行。不保存到历史记录中。 |

大多数工单几乎不会停留在 **Open** 状态。来自计划的维护工单会在一步内被创建并安排。在现场创建的故障工单会立即由技术员创建并安排。**Open** 是指管理员创建了工单但还没有选择技术员或日期时您所看到的状态。

工单在关闭之前可以随时被**取消** — 即使在客户签字之后,如果管理员需要作废它的话。**Closed** 和 **Cancelled** 是最终状态。

签字始终需要楼宇代表的签名。管理员不能代客户签字。

如果有需要重做的事情,管理员可以将工单退回一个阶段 — 例如从 **Signed** 退回到 **Work Done**。

## 五步工作流程

每种工单类型都遵循相同的五个步骤:

1. 工单被创建并分配给一名技术员 — 它显示为 **Scheduled**。
2. 技术员到现场并完成工作。
3. 技术员将工单标记为 **Work Done**。
4. 楼宇代表在工单上签字 — 工单变为 **Signed**。
5. 管理员审核并关闭工单 — 它变为 **Closed** 并保存到电梯的历史记录中。

不同类型之间的区别在于**工单如何被创建**以及**技术员在现场记录什么**。

## 工单类型

### Maintenance

例行的定期服务。技术员在现场逐项完成一份检查清单 — 每项通过或失败,必要时附上备注和照片。

维护工单通常由电梯的维护计划自动创建。该计划设置周期 — **monthly** 或 **quarterly** — 以及首选的日期和时间窗口。管理员也可以手动创建一个用于临时服务。

### Breakdown

对故障电梯的非计划访问,由楼宇管理员或居住者报告故障时触发。使用自由形式的备注和照片,而不是检查清单。

故障工单有两种创建方式:

- **Technician on-site** — 技术员到达电梯并自行创建工单。他们不等待管理员。
- **Admin dispatch** — 管理员在控制台中创建工单并派遣一名技术员。

如果故障无法当场修复,管理员通常会创建一个单独的维修工单作为后续。这是一个手动步骤 — 故障工单不会自动创建维修工单。

### Repair

为修复之前发现的问题而安排的访问,通常发生在故障期间,有时发生在维护期间。备注和照片描述所做的工作。

维修工单通常由管理员安排,特别是在需要订购零件时。技术员也可以从应用中创建一个。

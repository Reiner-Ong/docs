# 工作

一个工作是对一部电梯的一项工作。每个工作都属于特定的电梯,有一个类型,并经历相同的步骤从开始到完成。一旦关闭,它就成为电梯永久历史记录的一部分。

## 什么是工作

工作是对特定电梯的工作单元。每个工作:

- 属于一部电梯。
- 有一个类型 — **维护**、**故障** 或 **维修**。
- 经历与其他所有工作相同的步骤。
- 一旦安排好,就分配给一名技术员。
- 关闭后成为电梯历史记录的一部分。

已取消的工作不会保存到电梯的历史记录中。

## 工作如何流转

工作经过五个步骤:

**Open → Scheduled → Work Done → Signed → Closed**

| 步骤 | 含义 |
| --- | --- |
| Open | 已创建,但还没有技术员或日期。 |
| Scheduled | 已设置技术员和时间。准备开始工作。 |
| Work Done | 技术员已完成现场工作。等待客户签字。 |
| Signed | 建筑代表已在工作单上签字。 |
| Closed | 管理员已审核并关闭工作。现在是电梯历史记录的一部分。 |
| Cancelled | 工作不会发生。不保存到历史记录中。 |

大多数工作在 **Open** 状态下停留时间很短。来自计划的维护工作在一个步骤中被打开并安排。在现场创建的故障工作立即在技术员身上被打开并安排。**Open** 是当管理员创建了工作但尚未选择技术员或日期时您所看到的状态。

工作可以在关闭之前的任何时候被**取消** — 即使在客户签字之后,如果管理员需要作废它。**Closed** 和 **Cancelled** 是最终状态。

签字始终需要建筑代表的签名。管理员不能代表客户签字。

如果某些内容需要重做,管理员可以将工作退回一个阶段 — 例如从 **Signed** 退回到 **Work Done**。

## 五步工作流程

每种工作类型都遵循相同的五个步骤:

1. 工作被创建并分配给一名技术员 — 显示为 **Scheduled**。
2. 技术员到现场并完成工作。
3. 技术员将工作标记为 **Work Done**。
4. 建筑代表在工作单上签字 — 工作变为 **Signed**。
5. 管理员审核并关闭工作 — 它变为 **Closed** 并保存到电梯的历史记录中。

不同类型之间变化的是**如何创建工作**和**技术员在现场记录什么**。

## 工作类型

### 维护

例行的定期服务。技术员在现场完成检查表 — 每项通过或失败,并在需要时附上备注和照片。

维护工作通常由电梯的维护计划自动创建。计划设定频率 — **每月**或**每季度** — 加上首选日期和时间窗口。管理员也可以为临时服务手动创建一个。

### 故障

对故障电梯的计划外访问,由建筑经理或居住者报告故障时触发。使用自由格式的备注和照片,而不是检查表。

故障工作有两种创建方式:

- **技术员在现场** — 技术员到达电梯处并为自己创建工作。他们不需要等待管理员。
- **管理员派遣** — 管理员在仪表板中创建工作并派遣技术员。

如果故障无法当场修复,管理员通常会创建单独的维修工作作为跟进。这是一个手动步骤 — 故障不会自动创建维修。

### 维修

对早期发现的问题进行计划访问以修复,通常是在故障期间,有时是在维护期间。备注和照片描述已完成的工作。

维修工作通常由管理员安排,尤其是当需要订购零件时。技术员也可以通过应用程序创建一个。

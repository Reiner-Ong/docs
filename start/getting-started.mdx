LiftAuth is a lift maintenance management platform built for Singaporean lift servicing companies. It helps you schedule service visits, record completed work, track faults, and send invoices — all in one place.

There are three types of people in LiftAuth:

- **Admins** — office staff who plan and oversee operations from the web dashboard
- **Technicians** — field workers who carry out jobs on-site using the mobile app
- **Building Managers** — the people responsible for a building, who receive jobsheets to sign via email (no login required)

---

## Admin access levels

Not all admins have the same access. There are two admin roles in LiftAuth:

<div style={{ marginTop: "-1rem", marginBottom: "-1rem" }}>
  | Role      | What they can do                                                                                           |
  | --------- | ---------------------------------------------------------------------------------------------------------- |
  | **Owner** | Full access — manage the team, billing, invoices, and all settings,<br />is both a Technician and an Admin |
  | **Admin** | Manage jobs, invoices, and team members (cannot change billing)                                            |
</div>

<Tip>
  _Every company starts with one Owner. The Owner can then invite additional Admins and Technicians from the Team page._
</Tip>

## Technician access

Technicians mostly use the mobile app and can only see the jobs assigned to them. They cannot access some web dashboard features such as invoices, payments, or team settings.

<Warning>
  If a Technician needs access to the web dashboard features, they can be invited as an **Admin** as well, vice-versa. 
</Warning>
